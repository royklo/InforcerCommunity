Replace the current `$isExcludedSetting` scriptblock (lines 99-110) with:

```powershell
# ── Helper: check if a setting should be excluded by name or value ──
$isExcludedSetting = {
    param([string]$Name, [string]$Value = '')
    foreach ($excluded in $excludedSettingNames) {
        if ($Name -eq $excluded) { return $true }
    }
    foreach ($pattern in $appIdSettingPatterns) {
        if ($Name -match $pattern) { return $true }
    }
    # Structural noise — array count rows (ported from IntuneLens STRUCTURAL_NOISE)
    if ($Value -match '^\d+ items$') { return $true }
    # Value-based exclusions: standalone GUIDs (D-04: values only, not names),
    # group collection header, etc. (D-03)
    foreach ($pattern in $excludedValuePatterns) {
        if ($Value -match $pattern) { return $true }
    }
    return $false
}
```

Key points:
- `$Value = ''` makes the parameter optional (backward compatible, per D-02)
- Use `-match` (not `-cmatch`) to keep case-insensitive matching (handles uppercase GUIDs per Pitfall 3)
- Structural noise `^\d+ items$` is handled as a separate named check inside the scriptblock (matching IntuneLens STRUCTURAL_NOISE separation, per Research open question recommendation)
- GUID filtering applies to values only, not names (per D-04)
