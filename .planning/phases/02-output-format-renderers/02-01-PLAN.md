    **ConvertTo-InforcerDocJson.ps1** (per D-13, D-14, D-15, D-22, D-23, JSON-01, JSON-02):
    Create private function `ConvertTo-InforcerDocJson` with:
    - `[CmdletBinding()]` and comment-based help (`.SYNOPSIS`, `.DESCRIPTION`, `.PARAMETER DocModel`, `.OUTPUTS`)
    - Single parameter: `[Parameter(Mandatory)][hashtable]$DocModel`

    **ConvertTo-InforcerDocCsv.ps1** (per D-13, D-14, D-15, D-24, D-25, D-26, CSV-01, CSV-02):
    Create private function `ConvertTo-InforcerDocCsv` with:
    - `[CmdletBinding()]` and comment-based help
    - Single parameter: `[Parameter(Mandatory)][hashtable]$DocModel`
    - Build a `[System.Collections.Generic.List[object]]` of flat rows
    - Loop: `foreach ($prodName in $DocModel.Products.Keys)` -> `foreach ($catName in $product.Categories.Keys)` -> `foreach ($policy in $policies)` -> `foreach ($setting in $policy.Settings)`
    - Per setting row, create `[PSCustomObject]@{ Product = $prodName; Category = $catName; PolicyName = $policy.Basics.Name; SettingName = $setting.Name; Value = if ([string]::IsNullOrEmpty($setting.Value)) { '' } else { $setting.Value }; Indent = $setting.Indent; IsConfigured = $setting.IsConfigured }`
    - Add each PSCustomObject to the List
    - Convert: `($rows | ConvertTo-Csv -NoTypeInformation) -join [Environment]::NewLine`
    - Return single string
    - Assignments and Basics are NOT included in CSV (per D-26)
