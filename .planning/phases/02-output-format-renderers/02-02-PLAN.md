  <action>
    Create `module/Private/ConvertTo-InforcerMarkdown.ps1` with private function `ConvertTo-InforcerMarkdown` (per D-13, D-14, D-15, D-19, D-20, D-21):

    **Function signature:**
    - `[CmdletBinding()]` with comment-based help (`.SYNOPSIS`, `.DESCRIPTION`, `.PARAMETER DocModel`, `.OUTPUTS`)
    - Single parameter: `[Parameter(Mandatory)][hashtable]$DocModel`
    - Returns: string (Markdown text)

    **Private helper inside the file** -- `ConvertTo-MarkdownTable`:
    - Accepts `-Headers` (string array) and `-Rows` (array of string arrays)
    - Builds GFM pipe-delimited table: header row, separator row (`| --- | --- |`), data rows
    - Escapes pipe characters in cell values: `$cell -replace '\|', '\|'`
    - Replaces null/empty cell values with em dash `—` (per D-10)
    - Returns string (the complete table block)

    **Private helper** -- `ConvertTo-MarkdownAnchor`:
    - Accepts `-Text` (string)
    - Returns GitHub-flavored anchor: lowercase, spaces to hyphens, strip non-alphanumeric except hyphens
    - Pattern: `($Text.ToLower() -replace '[^a-z0-9\s-]', '' -replace '\s+', '-').Trim('-')`

    **Document structure (built with StringBuilder for consistency):**
    1. Header: `# Tenant Documentation: $($DocModel.TenantName)` + blank line + `*Generated: $($DocModel.GeneratedAt.ToString('yyyy-MM-dd HH:mm:ss')) UTC*` + blank line + if BaselineName not empty: `*Baseline: $($DocModel.BaselineName)*` + blank line
    2. TOC section: `## Table of Contents` + blank line + for each product: `- [$prodName](#$anchor)` + for each category within product: `  - [$catName](#$catAnchor)` (2-space indent)
    3. Separator: `---` + blank line
    4. For each product: `## $prodName` heading + blank line
    5. For each category: `### $catName` heading + blank line
    6. For each policy: `#### $($policy.Basics.Name)` heading + blank line
    7. Basics table: headers `Property`, `Value`; rows from Basics hashtable keys (Name, Description, ProfileType, Platform, Created, Modified, ScopeTags) -- skip Name since it is the heading
    8. Settings table (if $policy.Settings.Count > 0): headers `Setting`, `Value`; for each setting row: if `$setting.Indent -gt 0`, prefix Name with `$('  ' * $setting.Indent)↳ ` (per D-08); value from `$setting.Value`
    9. Assignments table (if $policy.Assignments.Count > 0): headers `Group`, `Filter`, `Filter Mode`, `Type`; rows from assignment properties
    10. Blank line between sections

    **No file I/O, no API calls.** Return the complete Markdown string.
  </action>
